Falling Catcher - Explained

Chapter 10 · Collision Detection + Bounce + Game Physics
CONCEPT 1

📦 Collision Detection

Every game object is really just an invisible rectangle (a "box"). To check if two things are touching, we compare their boxes: their left/right edges and top/bottom edges.

If all four of these are true, the boxes overlap:

  • Box A's left edge is before Box B's right edge
  • Box A's right edge is after Box B's left edge
  • Box A's top edge is before Box B's bottom edge
  • Box A's bottom edge is after Box B's top edge
function isColliding(a, b) { return a.x < b.x + b.w && a.x + a.w > b.x && a.y < b.y + b.h && a.y + a.h > b.y; }
CONCEPT 2

🏓 Bounce

A "bounce" is just a direction flip. Every moving object has a velocity - a speed and direction, often split into vx (left/right speed) and vy (up/down speed).

When the object hits a wall, we multiply that velocity by -1. Same speed, opposite direction!

if (ball.x <= 0 || ball.x + ball.w >= stageWidth) { ball.vx *= -1; // flip left/right direction }
CONCEPT 3

⚙️ Game Physics Loop

Games don't move things all at once - they update a tiny bit, over and over, many times per second. That's the game loop.

1

Move every object a little bit, based on its velocity.

2

Check for collisions (did anything touch a wall, basket, or another object?).

3

React to what happened (bounce, catch, remove, add score).

4

Draw everything in its new position, then repeat - forever, ~60 times a second!

function gameLoop() { moveEverything(); checkCollisions(); requestAnimationFrame(gameLoop); // do it again next frame }
