Breakout - Explained

Chapter 10 · Collision Detection + Bounce + Game Physics
CONCEPT 1

📦 Collision Detection

In Breakout, the ball is always checking: "am I touching a wall? the paddle? a brick?" Every object is really an invisible rectangle (a "box"), and we compare box edges to know if two things overlap.

function isColliding(ball, box) { return ball.x < box.x + box.w && ball.x + ball.size > box.x && ball.y < box.y + box.h && ball.y + ball.size > box.y; }

If this is true, the ball has just hit that box - time to react!

CONCEPT 2

🏓 Bounce

The ball has a velocity: vx (left/right speed) and vy (up/down speed). A bounce is just flipping one of those - multiplying it by -1.

  • Hit a side wall → flip vx
  • Hit the top wall, a brick, or the paddle → flip vy
if (ball.x <= 0 || ball.x + size >= width) { ball.vx *= -1; // side wall bounce } if (ball.y <= 0) { ball.vy *= -1; // top wall bounce }
CONCEPT 3

⚙️ Game Physics Loop

Breakout updates the ball's position a tiny bit, checks for collisions, reacts, and redraws - over and over, ~60 times a second. That's the game loop.

1

Move the ball by its velocity: x += vx, y += vy.

2

Check collisions against walls, the paddle, and every brick.

3

React: bounce the ball, remove a broken brick, add to the score.

4

Redraw everything in its new spot, then repeat with requestAnimationFrame.

function gameLoop() { moveBall(); checkWallCollisions(); checkPaddleCollision(); checkBrickCollisions(); draw(); requestAnimationFrame(gameLoop); }
